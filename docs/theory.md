# 数理理论说明

## 项目概述

本项目用于根据 EOT20 全球潮汐模型自动生成 ADCIRC 模型 `fort.15` 文件所需的潮汐开边界强迫数据。

主要流程包括：

1. 读取 ADCIRC 网格文件（fort.14）中的开边界节点；
2. 从 EOT20 潮汐模型读取各分潮调和常数；
3. 将潮汐信息插值到开边界节点；
4. 计算 ADCIRC 所需的天文节点因子（FF）和平衡角（FACE）；
5. 进行相位约定转换；
6. 输出可直接写入 fort.15 的潮汐强迫块。

---

# 潮汐调和分析基本原理

现代潮汐预报通常采用调和分析方法，将海面高度表示为多个天文分潮的叠加：

[
\eta(t)=\sum_{k=1}^{N} A_k f_k
\cos(\omega_k t + V_k + u_k - \kappa_k)
]

其中：

| 符号         | 含义         |
| ---------- | ---------- |
| (A_k)      | 分潮振幅       |
| (\omega_k) | 分潮角频率      |
| (f_k)      | 天文节点因子（FF） |
| (V_k+u_k)  | 平衡角（FACE）  |
| (\kappa_k) | 地方迟角（相位）   |

ADCIRC 在计算开边界潮位时采用上述形式，因此需要为每个分潮提供对应参数。

---

# EOT20 潮汐模型数据

EOT20（Empirical Ocean Tide Model 2020）是一套基于卫星测高资料构建的全球经验潮汐模型。

对于每个分潮，EOT20提供复数形式的调和常数：

[
H = H_{Re}+iH_{Im}
]

其中：

* (H_{Re})：实部
* (H_{Im})：虚部

通过复数形式可计算得到：

振幅：

[
A = |H|
]

相位：

[
\phi = \arctan\left(\frac{H_{Im}}{H_{Re}}\right)
]

本项目采用实部和虚部分别插值的方法，然后重建复数调和常数，以避免直接插值相位角时可能出现的跳变问题。

---

# 开边界插值

EOT20 数据存储于规则经纬度网格上，而 ADCIRC 开边界节点位置通常不与其重合。

因此需要将潮汐调和常数插值至开边界节点：

[
H_{node}=Interp(H_{grid})
]

程序优先采用双线性插值。

当节点位于陆海边界附近导致插值失败时，则自动采用最近邻插值作为补充。

---

# 天文节点因子（FF）

月球轨道并非固定不变，其轨道平面存在约18.61年的交点周期（Nodal Cycle）。

这一周期会引起潮汐振幅的长期变化。

因此实际振幅应表示为：

[
A'=fA
]

其中：

* (A) 为调和常数振幅；
* (f) 为节点因子（FF）。

FF通常接近1，但会随时间缓慢变化。

若忽略FF，在长时间模拟中可能产生明显误差。

本项目根据模拟起始时刻计算各分潮对应的FF值。

---

# 平衡角（FACE）

除了振幅变化外，天文运动还会引起潮汐相位缓慢变化。

调和分析中将其表示为：

[
V+u
]

ADCIRC中对应参数称为：

[
FACE
]

FACE与模拟参考时间直接相关。

当模型起始时间改变时，FACE也需要重新计算。

本项目依据 Schureman 潮汐天文理论和 NOAA 潮汐预报算法计算各分潮 FACE 值。

---

# 相位约定转换

不同潮汐模型和数值模型对于相位定义可能采用不同约定。

EOT20提供的复数相位与ADCIRC开边界所需相位之间存在符号和参考方向差异。

在实际应用中，半日分潮（特别是 M2、N2）通常需要进行180°转换：

[
\kappa_{ADCIRC}
===============

(\phi+180^\circ)
\bmod 360^\circ
]

本项目默认保留这一转换，以保持与既有ADCIRC工程配置的一致性。

如用户确认其数据集采用其他相位定义，可关闭该选项。

---

# ADCIRC潮汐开边界格式

ADCIRC 的 fort.15 文件要求为每个分潮提供：

* 分潮名称（BOUNTAG）
* 角频率（AMIG）
* 节点因子（FF）
* 平衡角（FACE）

随后提供每个开边界节点对应的：

* 振幅（Amplitude）
* 相位（Phase Lag）

本项目最终输出的数据块可直接复制至 fort.15 使用。
